# `slide`コマンド: Marpスライド生成

## 概要
このコマンドは、指定されたトピックに基づいてMarp形式のプレゼンテーションスライドを生成します。

## 実行プロセス

### 1. トピックの分析と構成案の提示
ユーザーから与えられた `{{topic}}` を分析し、まず **構成案のみ** を提示してください。

#### 構成案の出力フォーマット（固定）
構成案は、次のフォーマットで **14枚** を必ず提示してください（`templates/base-slide.md` 準拠）。

- 各行は `NN. [日本語タイトル] / [English Subtitle]` 形式
- NN は `01` から `14`
- この段階では **本文は書かない**（タイトルと意図だけ）
- 各スライドに「狙い（1行）」も併記する

**出力例（フォーマット例）**
```
01. タイトル / Lightning Talk
    - 狙い: 何の話で誰が話すかを即時に伝える
02. 今日話すこと / Agenda
    - 狙い: 全体像を先に共有して迷子を防ぐ
...
14. Q&A / Questions
    - 狙い: 質疑応答の時間を確保する
```

#### LT（ライトニングトーク）構成の採用
このプロジェクトでは、短時間で伝わる LT 形式の構成をデフォルトとします。

- 目安は **14〜15枚**（5〜7分想定）
- それぞれのスライドには「1枚1メッセージ」を徹底

構成案の中身は、最低限次の流れを満たしてください。

- タイトル
- アジェンダ
- 導入（問いかけ + 具体例）
- 結論
- 本論（背景・前提 → 全体像 → 具体ポイント）
- Before/Action/After（施策と効果）
- 応用（使いどころ）
- まとめ（Next Action）
- Q&A

> 参考: `templates/base-slide.md` に「base slide（14枚の型）」があります。構成案はこのテンプレと同じ並びにしてください。

#### 次のステップ（ユーザー承認の取り方）
- 構成案を提示したら、必ず「この14枚で進めてよいか／入れ替えたいスライドはあるか」を質問する
- 承認後にのみ、本文生成（ステップ2）へ進む

### 2. コンテンツ生成
構成案が承認されたら、各スライドの具体的なコンテンツを生成します。
- **マークダウン形式**: 出力は `GEMINI.md` のルールに従うこと。
- **言語**: 日本語を基本とします。
- **コード例**: 必要に応じて、適切な言語でシンタックスハイライト付きのコードブロックを挿入します。

#### スライド区切り
- 各スライドは必ず `<!-- スライド区切り -->` で区切ること（`GEMINI.md` のルール準拠）
- 14〜15枚のLTになるよう、スライド枚数を自己チェックして調整すること

#### LTの話運び（推奨）
- 導入→結論→根拠（本論）→応用→まとめ の順に、聞き手が迷子にならない導線にする
- 数値や主張には、可能なら根拠（前提/比較軸/条件）を添える

#### デザインテンプレート（sample_slide.pdf 風）
可能な限り、以下のコンポーネントを使って「情報の階層化」「ラベルでの属性分け」「Before/Action/After フレーム」を表現してください。

- 章スライドや見出しスライドでは、冒頭に大きいナンバリングと英語サブタイトルを入れる
  - `<div class="slide-no">01</div>`
  - `<div class="subtitle-en">Overview</div>`
- 強調ラベル（用途に応じて使い分ける）
  - `<span class="label issue">ISSUE</span>`
  - `<span class="label vision">VISION</span>`
  - `<span class="label topic">TOPIC</span>`
  - `<span class="label tips">TIPS</span>`
- Before / Action / After の3カラムカード（必要なスライドで使う）
  ```html
  <div class="cards">
    <div class="card before"><div class="card-title">Before</div>...</div>
    <div class="card action"><div class="card-title">Action</div>...</div>
    <div class="card after"><div class="card-title">After</div>...</div>
  </div>
  ```
- 編集ガイド（書き方例/入力見本が必要な場合）
  - `<div class="guide">（書き方例）…</div>`

### 3. ファイル名の指定
最終的に生成されるファイル名は `slide-deck.md` としてください。

## ユーザーからの入力
このプロンプトは `{{topic}}` という変数を受け取ります。

**例:**
`/slide "AIがソフトウェア開発に与える影響"`
