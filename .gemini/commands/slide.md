# `/slide` コマンド: Marp スライド対話生成

## 概要

ユーザーから受け取った `{{topic}}` をもとに、対話形式でスライドを構成し、最終的に Marp 形式の Markdown を `slides/` ディレクトリに生成します。

---

## 実行方法

チャット欄で以下のように入力してください：
```
/slide "スライドのトピックやタイトル"
```

---

## フェーズ構成とAIの振る舞い

AI（あなた）は以下のステップに従って、ユーザーと対話しながら進めてください。

### Phase 1: 構成案の提示（骨格の合意）
- ユーザーの `{{topic}}` に基づき、14〜15枚程度のスライド構成案（タイトルのみのリスト）を提示します。
- ユーザーから構成案の承認を得るまで、Markdown の全文生成は行いません。

### Phase 2: ドラフト生成（ファイル書き出し）
- 構成案が承認されたら、**`slides/` ディレクトリ配下にファイルを書き出します。**
- ファイル名ルール: 英小文字・数字・ハイフンのみ（例: `future-ai-dev.md`）
- `write_file` を使用して保存してください。

### Phase 3: レビュー・修正の案内
ファイルを書き出したら、AI は必ず以下のメッセージを送り、**ユーザーの入力を待ちます。**

```
ドラフトを `slides/[filename].md` として作成しました。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 📝 ユーザー様へのお願い
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. エディタで `slides/[filename].md` を開き、内容を修正してください。
2. グラフやKPI等の装飾を追加したい場合は `templates/SNIPPETS.md` を参照してください。
3. 修正が終わりましたら、チャットで「ビルドして」などと送ってください。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Phase 4: ビルド・画像取得の実行
ユーザーから修正完了の合意（「ビルドして」「OK」など）を受け取ったら、以下のビルドコマンドを実行、または提示します。

```bash
# 画像取得 + ビルドを一括実行
bash scripts/build.sh slides/[filename].md --fetch
```
ビルドが成功したら、`dist/` ディレクトリに生成された PDF/HTML のパスを案内して終了します。
