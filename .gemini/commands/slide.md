# `/slide` コマンド: Marp スライド対話生成

## 概要

ユーザーから受け取った `{{topic}}` をもとに、**対話形式でスライドを育てながら**
Marp 形式の Markdown を生成するコマンドです。

---

## フェーズ構成

```
Phase 0: ヒアリング        ← 方向性を決める
Phase 1: 構成案の提示      ← 骨格の合意
Phase 2: ドラフト生成      ← slides/ ディレクトリにファイルを書き出す
Phase 3: レビュー・修正    ← ユーザーが slides/ 内のファイルを直接編集する（★重要）
Phase 4: ビルド・画像取得  ← 最終成果物（dist/）を出力する
```

---

## Phase 2: ドラフト生成（ファイル書き出し）

構成案が合意されたら、**`slides/` ディレクトリ配下にファイルを書き出します。**

### 2-1. ファイル名の確定
- ルール: 英小文字・数字・ハイフンのみ（例: `aws-saa-roadmap.md`）

### 2-2. Markdown の書き出し
`write_file` を使用し、`slides/[filename].md` として出力します。
※ `slides/` ディレクトリは Git 管理外の作業用スペースです。

---

## Phase 3: レビュー・修正（ユーザー主導）

ファイルを書き出したら、AI は以下のメッセージを送り、**ユーザーの入力を待ちます。**

```
ドラフトを `slides/[filename].md` として作成しました。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 📝 ユーザー様へのお願い
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. エディタで `slides/[filename].md` を開き、内容を修正してください。
2. 修正が終わりましたら、チャットで「ビルドして」などと送ってください。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

---

## Phase 4: ビルド・画像取得

ユーザーから指示を受け取ったら、ビルドコマンドを提示します。

```bash
# 画像取得 + ビルド（slides/ 内のファイルを指定）
bash scripts/build.sh slides/[filename].md --fetch
```
