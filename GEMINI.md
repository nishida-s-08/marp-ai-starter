# GEMINIコンテキスト: Marp AI Starter（プロジェクト前提・ルール）

このファイルは、AI（Gemini CLI / Claude Code など）がこのリポジトリで作業する際に守るべき **前提** と **運用ルール** をまとめたものです。

---

## 1. ディレクトリ構成と各ファイルの役割

リポジトリの安定運用のため、各ディレクトリの役割を以下のように定義します。

| ディレクトリ / ファイル | 役割 | Git管理 | 備考 |
|---|---|---|---|
| `.gemini/commands/` | AIカスタムコマンド定義 | 〇 | `/slide` などのプロンプトを管理 |
| **`slides/`** | **スライド用 Markdown 保存先** | **×** | AIがドラフトを出力する作業用ディレクトリ |
| `themes/` | スタイルシート（CSS） | 〇 | ビルド時に自動で読み込まれる |
| 　├ `base.css` | 基本デザイン（土台） | 〇 | **編集禁止**。標準レイアウトを定義 |
| 　├ `charts.css` | コンポーネント用スタイル | 〇 | **編集禁止**。グラフ・カード等の装飾 |
| 　└ **`project.css`** | **個別カスタマイズ** | 〇 | **ここだけ編集**。色・フォントの上書き用 |
| `templates/` | 素材集 | 〇 | 汎用的なテンプレートとスニペット |
| `scripts/` | 実行スクリプト | 〇 | `build.sh` (ビルド), `fetch-image.sh` (画像) |
| `assets/images/` | 画像アセット | **×** | 取得した画像が保存される場所 |
| `dist/` | ビルド成果物 | **×** | 生成された PDF / HTML の出力先 |
| **`documents/`** | **参考資料** | **×** | スライド生成時にAIが参照するPDFやテキスト |

## 2. スライド生成の基本フロー（AI用）

AIは以下のステップで作業を進めること。

1.  **Phase 0-1: ヒアリングと合意**
    - **必要に応じて `documents/` 内の参考資料を確認する。**
    - 構成案（タイトルのみ）が固まるまで、Markdown の生成は行わない。
2.  **Phase 2: ドラフト作成と保存**
    - `slides/` ディレクトリ配下に、合意したファイル名で保存する。
3.  **Phase 3: ユーザーレビュー（必須）**
    - ファイルを作成したら作業を中断し、ユーザーに `slides/` 内のファイル編集を促す。
4.  **Phase 4: ビルド実行**
    - ユーザーの修正完了を確認後、`scripts/build.sh` を使用してビルドを案内する。

## 3. デザイン・エンジニアリング・ガイドライン（デザイン強化ルール）

AIはスライド生成時、単なるMarkdown記法（# や -）に頼らず、以下の **ARK GRAPHICS コンポーネント** を積極的に組み込み、リッチな視覚効果を得ること。

### A. レイアウトの原則
- **2カラム構成の優先**: 情報を整理する際は必ず `<div class="cols">` または `<div class="cols-half">` を使用し、左右にテキストとビジュアル（画像・KPI・図解）を振り分けること。
- **背景クラスの使い分け**: スライドの役割に応じて `` を必ず冒頭に記述する。
  - `title-slide`: 表紙。ダークネイビー背景。
  - `section-slide`: 章の区切り。アクセント背景。
  - `standard`: 通常の白背景。

### B. 推奨コンポーネント（HTMLタグの直接出力）
AIは以下のHTML構造をMarkdown内に直接出力すること：
- **KPIカード**: 重要な数値や実績。
  `<div class="kpi-cards"><div class="kpi-card"><h3>ラベル</h3><div class="kpi-value">数値</div></div></div>`
- **ステップリスト**: 手順やフェーズの説明。
  `<div class="step-list"><div class="step-item">...</div></div>`
- **インサイト（💡）**: グラフや表の下に添える分析コメント。
  `<div class="data-insight">💡 分析結果をここに記述</div>`
- **ピルバッジ**: カテゴリ名や強調キーワード。
  `<span class="pill accent">Keyword</span>`

### カラートークン（:root 変数）
- `--color-primary`: 見出し・アクセント
- `--color-accent`: サブタイトル・Afterカード
- `--color-background`: 背景
- `--color-foreground`: テキスト

### プロンプト注入ルール
AIは `/slide` 実行時、常に `templates/SNIPPETS.md` の内容を内部的に参照し、最新のコンポーネント記法を反映したコードを出力すること。

## 4. 画像とグラフの技術仕様
- 画像: `<!-- fetch-image: ... -->` を使い、`assets/images/` 配下のファイルを参照。
- グラフ: `Chart.js` を使用し、色は `project.css` の CSS 変数から取得して動的に反映。
