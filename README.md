# Marp AI Starter

AIを活用して、高品質なMarpプレゼンテーションを効率的に生成・管理するためのテンプレートです。

## 概要

AIによる構成案の作成 → ドラフト生成 → ユーザーによる編集 → 自動ビルドという、スピードと品質を両立したワークフローを提供します。

---

## 事前準備

### 1. Marp CLI のインストール
スライドをPDFやHTMLに変換するために必要です。
```bash
npm install -g @marp-team/marp-cli
```

### 2. 環境設定（画像自動取得用）
Unsplashから画像を自動取得する場合は、`.env`ファイルにAPIキーを設定してください。
```bash
cp .env.example .env
# .env を開き UNSPLASH_ACCESS_KEY を設定
```

## スライド生成ワークフロー

### 1. AI による構成案とドラフトの生成
`documents/` に参考資料（PDFやテキスト）を配置しておくと、AIが内容を読み取ってスライドに反映します。

その後、チャットでトピックを指定してコマンドを実行します。
```bash
/slide "生成AIが変えるソフトウェア開発の未来"
```
AIが構成案（目次）を提示します。内容に合意すると、**`slides/`** ディレクトリに Markdown ファイルが書き出されます。

### 2. ユーザーによる修正（★重要）
生成された `slides/filename.md` をエディタで開き、**内容の確認と修正のみ**を行います。

**ユーザーがやること:**
- 文章・数値・固有名詞が正しいかチェックする
- 不要なスライドを削除する、または追加が必要な内容をAIに伝える

**ユーザーがやらなくていいこと（AIが自動処理):**
- グラフやKPIカードの挿入・選択（AIが自動組み込み済み）
- デザインコンポーネントの選択（AIが自動選択済み）
- 画像キーワードの設定（AIが `<!-- fetch-image: ... -->` を挿入済み）

### 3. 自動ビルド
修正が完了したら、ビルドスクリプトを実行して成果物（PDF/HTML）を出力します。
```bash
# 画像を自動取得してビルド（dist/ に出力）
bash scripts/build.sh slides/filename.md --fetch
```

## プロジェクト構成

```
marp-ai-starter/
├── .gemini/commands/   # AI用カスタムコマンド定義（/slide 等）
├── documents/          # [Git除外] スライド生成用の参考資料（PDF・テキスト）
├── slides/             # [Git除外] スライド作業場。AIがここに出力
├── themes/             # スタイルシート（ビルド時に自動統合）
│   ├── base.css        # 基本デザイン（編集禁止）
│   ├── charts.css      # コンポーネント用（編集禁止）
│   └── project.css     # ★個別カスタマイズ（色やフォントを変更）
├── templates/          # 素材集（スニペットや基本の型）
│   ├── base-slide.md   # AIが参照するスライド骨格テンプレート（ユーザーは直接触らない）
│   └── SNIPPETS.md     # AIが参照するコンポーネント素材集（ユーザーは直接触らない）
├── scripts/            # 実行ツール（ビルド、画像取得）
├── dist/               # [Git除外] 生成された PDF / HTML の出力先
├── GEMINI.md           # AIへの指示書（コンポーネント仕様など）
└── README.md           # このファイル
```

## テンプレートファイルについて（`templates/`）

`templates/` の2ファイルは**どちらもAIが内部的に参照する素材**です。ユーザーが直接編集する必要はありません。

### `base-slide.md` — AIが参照するスライドの骨格テンプレート

表紙・アジェンダ・課題・結論・グラフ・Q&Aなど、必要なスライドタイプが14〜16枚分そろった雛形です。AIが `/slide` コマンドを実行するとき、この構造を起点にトピックに合わせたドラフトを生成します。

### `SNIPPETS.md` — AIが参照するコンポーネント素材集

グラフ・KPIカード・データテーブルなど、視覚的なコンポーネントが12種類収録された素材集です。AIがドラフトを生成する際、スライドの内容に応じて**自動的に**適切なコンポーネントを選択・組み込みます。ユーザーがここからコピーする必要はありません。

| スニペット | 内容 | 主な用途 |
|---|---|---|
| SNIPPET 01 | KPIカード 4列 | 実績サマリー |
| SNIPPET 02 | データテーブル（達成率バッジ付き） | 目標vs実績 |
| SNIPPET 03 | 棒グラフ | 比較・達成率 |
| SNIPPET 04 | 折れ線 + ドーナツグラフ | トレンド + 比率 |
| SNIPPET 05 | 折れ線（エリア塗り）+ レーダー | 推移 + 多軸評価 |
| SNIPPET 06 | 画像 + テキスト 2カラム | 事例・紹介 |
| SNIPPET 07 | 背景画像スライド | ビジュアル強調 |
| SNIPPET 08 | 円グラフ + KPIカード | 構成比 + 数値 |
| SNIPPET 09 | プログレスバー + 数値ダッシュボード | 進捗・達成度 |
| SNIPPET 10 | 横棒グラフ（ランキング） | 順位・比較 |
| SNIPPET 11 | 複合グラフ（棒 + 折れ線） | 数量 + 比率の重ね合わせ |
| SNIPPET 12 | インサイト + タイムライン | 振り返り・展望 |

## カスタマイズ（project.css）

スライド全体の配色やフォントを変えたい場合は、**`themes/project.css`** を編集してください。
プリセット（Green, Orange, Purple等）がコメントアウト形式で用意されています。
