# 解決する課題

-   **時間のかかる手作業:**
    プレゼンテーション資料の作成に多くの時間と労力がかかる。
-   **デザインの一貫性:**
    ゼロからデザインを構築・維持するのが困難。
-   **「まっさらなページ」問題:**
    構成やコンテンツのアイデア出しに時間がかかり、手詰まり感が生じる。

# ソリューション: Marp AI Starter

AIとMarp CLIを組み合わせることで、**プレゼンテーション資料の作成プロセス全体を自動化**し、迅速かつプロフェッショナルなスライド生成を実現します。

# プロジェクトの目的とビジョン

-   **目的:**
    AIによる「発想支援」と「構造化」で、堅牢なデザインシステム（Marp）と統合し、誰でも簡単に高品質なプレゼン資料を作成できるようにする。
-   **ビジョン:**
    プレゼンテーション作成のハードルを劇的に下げ、アイデアの共有と伝達をより効率的かつ効果的にする。

# 主な機能

-   **AIによる構成案・コンテンツ生成:** トピックに基づき、14枚構成案と詳細コンテンツを自動生成。
-   **豊富なコンポーネント:** KPIカード、チャート、テーブルなどの定義済みコンポーネント。
-   **テーマベースのデザイン:** `project.css` で容易にカスタマイズ可能。
-   **自動画像取得:** Unsplashから関連画像を自動でダウンロード。
-   **マルチフォーマット出力:** PDF, HTML, PPTXへエクスポート。

---
class: feature-1
---

# 機能①：AIによる構成案・コンテンツ生成

-   **ワークフロー:**
    1.  `/slide 'トピック'` コマンドでAIに指示。
    2.  AIが14枚のスライド構成案を提示 → ユーザーが承認。
    3.  承認後、AIが各スライドの詳細コンテンツを生成。
-   **メリット:**
    *   「何から書けばよいか」の悩みを解消。
    *   論理的で網羅的な資料を素早く作成。

---
class: feature-2
---

# 機能②：コンポーネントライブラリとスニペット

-   **活用:**
    *   `.issue-block`, `.topic-cards`, `.vision-block` などの定義済みコンポーネントを使用。
    *   `GEMINI.md` (AIルールブック) と `SNIPPETS.md` (コードスニペット集) により、構造化されたコンテンツ生成を支援。
-   **メリット:**
    *   デザインに沿った一貫性のある情報表現。
    *   複雑な要素も容易にスライドに組み込める。

---
class: feature-3
---

# 機能③：テーマとカスタマイズ

-   **デザインシステム:**
    *   `themes/base.css` で基本スタイルを定義。
    *   `themes/project.css` でプロジェクト固有の色やフォントを上書き。
-   **カスタマイズ性:**
    *   ブランドガイドラインや好みに合わせたデザイン調整が容易。
    *   Marp CLIによる多様な出力形式に対応。

# 自動画像取得機能

-   Markdownファイル内のコメントに基づき、AIがUnsplashから関連画像を自動で検索・ダウンロードします。
-   プレゼンテーションの視覚的な魅力を、手作業なしで高めることができます。

# アーキテクチャと技術スタック

-   **AIレイヤー:** Gemini CLI / プロンプトエンジニアリング (`.gemini/commands/slide.md`)
-   **プレゼンエンジン:** Marp CLI
-   **スタイリング:** HTML, CSS (CSS Variables), `themes/base.css`, `themes/project.css`
-   **自動化:** Bash スクリプト (`scripts/build.sh`)

---
class: build-process
---

# ビルドプロセス

-   **PDF / HTML 生成:**
    ```bash
    bash scripts/build.sh slide-deck.md
    ```
    （`/dist` ディレクトリに出力）

-   **PPTX 生成:**
    ```bash
    marp slide-deck.md --pptx ...
    ```
    （`slide/` ディレクトリに出力）
